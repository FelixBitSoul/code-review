# Code Review Skill

专业的代码审查技能模块，对代码进行全面的规范性和质量评估，输出结构化分级报告。

## 📌 概述

这是一个用于进行高质量代码审查的技能模块，具备以下特点：

- **专业评估**：基于10+年软件工程经验的标准进行审查
- **多维度分析**：从正确性、安全性、性能、可读性、可维护性等多个角度评估
- **结构化报告**：自动生成清晰的分级问题报告（严重/一般/优化建议）
- **多语言支持**：支持 Python、Java、JavaScript 等多种编程语言
- **自动触发**：支持手动触发和自动触发两种模式

## 🎯 核心功能

### 代码审查维度

| 维度 | 描述 |
|------|------|
| **正确性** | 逻辑是否正确，边界条件是否处理完善 |
| **安全性** | 是否存在注入、越权、敏感信息泄露等风险 |
| **性能** | 是否存在明显的性能瓶颈或不必要的资源消耗 |
| **可读性** | 命名是否清晰，注释是否合理 |
| **可维护性** | 耦合度是否合理，是否遵循 SOLID 原则 |
| **测试覆盖** | 是否有测试，测试是否充分 |

### 问题分级

- 🔴 **严重问题**：影响功能正确性、存在安全漏洞、可能导致生产事故 → **必须修复**
- 🟡 **一般建议**：影响代码质量、可维护性，但不影响当前功能 → **建议修复**
- 🟢 **优化建议**：性能优化、代码简化等锦上添花的改进 → **可选**

## 🚀 使用方式

### 方式1：手动触发

在代码对话框中使用命令：

```
/code-review
```

然后粘贴你要审查的代码。

### 方式2：自动触发

当用户提及以下关键词时自动调用此技能：

- `code review`、`代码审查`、`审查代码`
- `检查代码`、`review this`
- `帮我看下这段代码`、`代码有没有问题`

### 示例对话

```
用户："帮我审查一下这段 Python 代码"
[粘贴代码]

Agent: 自动进行代码审查，输出分级报告
```

## 🔧 技术实现

### 核心文件说明

| 文件 | 用途 |
|------|------|
| `SKILL.md` | 定义技能的工作流程、审查维度和报告格式标准 |
| `resources/checklist.md` | 各语言的详细检查清单，指导审查过程 |
| `scripts/format_report.py` | Python 脚本，将结构化审查数据转换为美观的 Markdown 报告 |

### 工作流程

```
1. 读取 checklist.md（了解审查维度）
    ↓
2. 逐条分析用户提交的代码
    ↓
3. 调用 format_report.py 格式化报告
    ↓
4. 输出结构化分级报告
```

## 💡 最佳实践

### 代码审查者应该

1. **保持客观**：基于规范而非个人偏好进行评估
2. **重点突出**：优先关注严重问题，其次是可维护性
3. **给出建议**：不仅指出问题，还要给出可行的修复方案
4. **鼓励积极**：肯定代码中做得好的地方
5. **考虑上下文**：理解业务背景和性能约束

### 开发者应该

1. **主动审查**：在提交 PR 前进行自审查
2. **乐于接受反馈**：审查意见是为了代码质量
3. **快速迭代**：根据审查意见及时修改
4. **学习沉淀**：总结常见问题，避免重复犯错

## 📈 评分标准

### 总体评分

| 评分范围 | 等级 | 含义 |
|---------|------|------|
| 9-10 | A | 优秀 - 代码质量高，可直接合并 |
| 7-8 | B | 良好 - 有少量问题需修复 |
| 5-6 | C | 中等 - 有明显问题需要重点改进 |
| <5 | D | 不合格 - 有严重问题，需大幅修改 |

## 📞 常见问题

**Q: 可以审查什么类型的代码？**  
A: 支持任何编程语言的代码。对于 Python、Java、JavaScript 等主流语言有专项检查项。

**Q: 审查需要多长时间？**  
A: 取决于代码复杂度。一般 100 行代码的审查需要几分钟。

**Q: 报告可以定制吗？**  
A: 可以。修改 `SKILL.md` 中的输出格式或 `checklist.md` 中的检查项即可。
